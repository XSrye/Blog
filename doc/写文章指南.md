# 写文章及发布操作指南

## 目录结构

```
src/content/blog/<文章目录名>/
  └── zh-cn.md      # 文章内容（Markdown）
```

每篇文章一个目录，目录名用英文短横线命名（如 `my-new-article`）。

---

## 一、创建新文章

### 方式 1：使用脚本（推荐）

```bash
cd /opt/Momo
npm run newpost <目录名>
```

例如：

```bash
npm run newpost my-new-article
```

会在 `src/content/blog/my-new-article/` 下创建 `zh-cn.md`。

### 方式 2：手动创建

在 `src/content/blog/` 下新建目录和文件：

```
src/content/blog/
  └── my-new-article/
      └── zh-cn.md
```

---

## 二、文章 Frontmatter 格式

每篇文章头部需包含以下元信息：

```yaml
---
title: "文章标题"
pubDate: 2026-07-08
description: "文章简介，会显示在列表页和 RSS 中"
category: 分类名       # 可选，如：杂文、杂谈、技术、生活
image: ""              # 封面图路径，留空则无封面
draft: false           # true=草稿不发布，false=发布
slugId: "my-unique-id" # 唯一标识，用于 URL 和评论（可省略）
---
```

**注意：**
- `pubDate` 是发布日期，格式 `YYYY-MM-DD`
- `category` 会显示在文章页的分类标签中
- `draft: true` 时文章不会出现在列表页，但可以通过 URL 直接访问

---

## 三、文章内容写法

Frontmatter 下方写正文，支持 Markdown 语法。

### 示例

```markdown
---
title: "我的第一篇文章"
pubDate: 2026-07-08
description: "这是我的第一篇博客文章"
category: 杂谈
draft: false
---

## 介绍

这里是正文内容，支持 **加粗**、*斜体*、`代码` 等。

### 列表

- 项目一
- 项目二

### 代码块

```python
print("Hello, World!")
```

### 图片

![](/upload/图片文件名.png)
```

### 图片上传

图片存放在 `/opt/Momo/public/upload/` 目录下，然后在文章中引用：

```markdown
![](/upload/图片名.png)
```

## 四、本地预览

```bash
cd /opt/Momo
npm run dev
```

浏览器打开 `http://localhost:4321` 即可实时预览。

---

## 五、发布到线上

### 方法一：手动推送

```bash
cd /opt/Momo
git add -A
git commit -m "添加新文章：文章标题"
git push
```

推送后 Cloudflare Pages 会自动构建部署（约 1-2 分钟）。

### 方法二：直接在 GitHub 网页端

1. 打开 https://github.com/XSrye/Blog
2. 找到 `src/content/blog/` 目录
3. 点击 **Add file** → **Create new file**
4. 按上述格式写文章
5. 提交 PR 到 `main` 分支 → Cloudflare 自动部署

---

## 六、验证部署

- **博客地址**：https://blog.pages.dev
- **RSS 订阅**：https://blog.pages.dev/rss.xml

推送后等 1-2 分钟，访问博客确认新文章已显示。
